Loop

Use [loop] to get posts and loop through them.

Display five most recent posts

[loop type="post" count="5"]   [field title]   [field date]   [field excerpt] [/loop]

Display all posts in a custom post type and category

[loop type="apartment" category="suite"]   Apartment: [field title]   Rent per day: [field price]   Description: [content] [/loop] The [content] and [field] shortcodes target each post in the loop.

Parameters

Post type

  • type - post, page, or custom post type - if empty, default is any; multiple values possible

Post ID

  • id - show post by ID; id="1,3,5,7"
  • exclude - exclude post by ID; exclude="1,3,5,7"
    • this - exclude current post
    • children - show only top-level posts

Post count and status

  • count - number of posts to show - default is all
  • offset - offset the loop by a number of posts, for example: skip the first 3 posts
  • status - display posts by status: any, publish, pending, draft, future, private; the default is publish - multiple values possible, i.e., status="publish, pending"

Author

  • author - show posts by author ID, login name; multiple values possible; use this for current user, and same for the same author as current post
  • author_exclude - exclude posts by author ID, login name, or this for current user
  • role - show posts by user role, such as administrator and editor. Use role="this" to get posts by authors in the same role as current user

Published date

  • year, month, day - display posts by specific year, month, day; you can use today as a value, for example month="today" will show posts from this month
  • before - display posts before a relative or specific date, for example: 2 weeks ago, or 2015-02-01
  • after - display posts after a date

Category, tags and taxonomies

  • category - display posts from one or more categories: category="sports, fashion"
  • tag - display posts with one or more tags: tag="apples, green"
  • compare - for multiple categories or tags, set compare="and" to get posts that have all terms
  • taxonomy, term - display posts by taxonomy term: taxonomy="product" term="book"
  • compare - use compare="not" for posts whose taxonomy is not term; use compare="and" when you want posts whose taxonomy contains all terms; default is compare="or" to get posts whose taxonomy contains any of the given terms

    Multiple taxonomy queries

  • relation - additional taxonomy query, where relation is and (default) or or
  • taxonomy_2, term_2, compare_2...

Field value

  • field - field name
  • value - value to search for
  • start - use instead of value to check only the beginning of field value
  • compare - equal (default), not, more, less, or operator like < and >.
  • compare="between" - query for a range of values; for example, value="0,100"

    Multiple field queries

  • relation - additional field query, where relation is and (default) or or
  • field_2, value_2, compare_2...

Date field

  • field - field name
  • value - field value
    • You can specify exact value, or use the following predefined values.
    • today - compare to today
    • now - if your field contains date and time
    • future - today and after
    • past - before today
    • past and today - past including today
  • compare - equal (default), not, more, less, or operator like < and >.
  • before, after - used in place of value and compare; query for field values before/after a relative or specific date: 2 weeks ago, or 2015-02-01
  • date_format - date format of the field value; default is "Y-m-d". For ACF date field, set it to "Ymd".

    Multiple date fields

  • field_2, value_2, compare_2, date_format_2, after_2, before_2...

Parent

  • parent - display all children of a parent specified by ID or slug
    • this - get current post's children
    • same - get current post's siblings (posts which share the same parent)
  • exclude
    • children - display top-level posts only
    • this - don't include current post

Sorting and series

  • orderby - order by ID, author, title, name, date, comment-date, rand (randomized), modified, menu_order, parent, field (field value: string), or field_num (field value: number)
  • order - ASC (ascending/alphabetical) or DESC (descending/from most recent date) when using the orderby parameter
  • key - when ordering by field or field_num, specify key as the name of the custom field to order by
  • series, key - order posts by a series of custom field values, where key is the name of the field - the series can include ranges, for example: 1-15,30-40,42,44.

Checkboxes

  • checkbox, value - display posts according to custom field, whose checkbox contains the value
    • Multiple values are possible: value="first, second". This will return all posts with checkboxes containing any of the values, i.e., first or second. Optionally, you can set the parameter compare="and", which will return all posts with checkboxes containing all of the values, i.e., first and second. See example section below.
  • relation, checkbox_2, value_2 - additional query for checkbox value, where relation is AND (default) or OR. You can use multiple values, and the parameter compare_2, if necessary.

Formatting

  • trim - set true to remove space or comma at the end
  • clean - set true to remove all <p> and <br> tags
  • strip_tags - set true to remove all HTML tags inside the loop
  • allow - strip all HTML tags except allowed, for example: allow="<a><li><br>"

List

  • list - set true to create a list with ul; or specify tag like ol or div
  • list_class, list_style - add class or style to the list
  • item - tag to wrap each loop item; default is li, or specify tag like span
  • item_class, item_style - add class or style to each item

Paged

  • paged - number of posts per page
  • maxpage - maximum number of pages
    • These are used with the [loopage] shortcode to create pagination.

Cache

  • cache - set true to cache the result of the loop; see here for details
  • expire - how often the cache is updated: minutes, hours, days, years - default is 10 minutes
  • update - set true to force update the cache

Other parameters

  • name - show one post by slug
  • blog - switch to given blog ID on multi-site
  • columns - display output in columns: columns="3"; use pad for padding: pad="0px 10px"
  • timer - set true to display resource info at the end of loop; see here for details
  • x - repeat the loop x times - no query

Examples

Query for field values

Single field

[loop type="hotels" field="stars" value="4"]

Single field compare

[loop type="hotels" field="stars" compare="not equal" value="3"]

Single field: if value exists

[loop type="hotels" field="event_date" compare="exists"]

Two field values

[loop type="hotels" field="stars" value="4" field_2="service" value_2="all inclusive"]

Two field values relation

[loop type="hotels" field="stars" value="4 stars" relation="or" field_2="service" value_2="all inclusive"]

Series

[loop type="books" series="1,2,3,7-15,50-60" key="serial_number"]

Query by date field

Show future events; field is stored as timestamp

[loop type="events" field="event_date" value="future"]

Show future events, when the field is stored as a string, i.e., 2015-03-12

[loop type="events" field="event_date" value="future" in="string"]

Show future events, when the field is a string with date and time, i.e., 2015-03-12 13:00

[loop type="events" field="event_date" value="future-time" in="string"]

Show future events in descending order

[loop type="events" field="event_date" value="future" orderby="field_num" order="DESC"]

Query by checkbox values

Single checkbox

[loop type="movies" checkbox="genre" value="comedy"]

Single checkbox and multiple values: sci-fi or fantasy

[loop type="movies" checkbox="genre" value="sci-fi, fantasy"]

Single checkbox and multiple values: comedy and drama

[loop type="movies" checkbox="genre" compare="and" value="comedy, drama"]

Two checkboxes..

[loop type="movies" checkbox="genre" compare="and" value="comedy, drama" checkbox_2="in-stock" value_2="current, pending"]

Field tags

This is a feature to expand a list of fields to their values. [loop type="post" fields="custom_field, another_field"]   Display {CUSTOM_FIELD} and {ANOTHER_FIELD} [/loop] The {FIELD} tags are uppercase versions of the field names. There are also predefined tags: COUNT, URL, ID, TITLE, AUTHOR, AUTHOR_URL, DATE, THUMBNAIL, THUMBNAIL_URL, CONTENT, EXCERPT, COMMENT_COUNT, CATEGORY, TAGS, IMAGE, IMAGE_ID, IMAGE_URL. If you want to pass field values to the loop shortcode itself, refer to: Field as parameter.
