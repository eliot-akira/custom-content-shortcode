User info

Display current user info

ID: [user id] Full name: [user] E-mail: [user email]

User fields

  • id - ID
  • name - login name
  • email - e-mail
  • url - website URL
  • avatar - avatar image
  • size - avatar image size in pixels - default 96, max 512
  • url - website URL
  • role - user role
  • post-count - user post count
You can also specify a custom user field. [user field_name]

User field value

Use [if] to check that a user field is not empty, or has specific value.

If user field is not empty

[if user_field="facebook_profile"]     <a href="[user facebook_profile]">Facebook profile</a> [else]     No Facebook profile [/if]

If user has posts

[if user_field="post-count"]     Post count: [user post-count] [else]     No posts yet! [/if]

If user field has specific value

[if user_field="school" value="Home Town University"]     Hey, schoolmate! [/if]

Users loop

Make a list of admin users

[users role="admin"]     Admin: [user]     Contact: [user email] [/users]

Parameters

  • role - admin, editor, author, contributor, subscriber
  • include - include users by ID
  • exclude - exclude users by ID
  • orderby - ID, display_name, name, login, email, url, registered, post_count, field, field_num
  • order - ASC - alphabetical (default) or DESC (new to old)
  • number - maximum number of returned results
  • offset - offset the results by a number
  • field - custom field name to query
  • value - field value; multiple values possible depending on compare
  • compare - equal (default), not, in, not in, between, not between, or operators like >= and <=.
  • search - search for string match on user columns
  • search_columns - one or more columns to search: ID, login, nicename, email, url
  • blog_id - blog ID on a multisite

Sort by user field

Field value is string

[users orderby="field" field="twitter"]

Field value is number

[users orderby="field_num" field="position"]

User status

The [is] shortcode is used to display content based on user status.

Display user status

[is admin]You are an administrator.[/is] [is author]You are the author of current post.[/is] [is login]You are logged in.[/is] [is logout]You are logged out.[/is] [is user="john"]You are John.[/is] [is role="subscriber"]You are a subscriber.[/is]

Parameters

  • admin - user has admin capability (manage options)
  • author - user is author of current post
  • login - user is logged in
  • logout - user is logged out
  • user - user name or ID - multiple values are possible: "1, 3, 22"
  • role - user role - default roles are administrator, editor, author, contributor, or subscriber
  • capable - user capability, for example: capable="manage_options"
    • Multiple values are possible: role="admin, subscriber" will be true if the user is admin or subscriber. See the list of available user roles and capabilities under Dashboard -> Content -> User Roles.
You can use [else] or [is not] to display something when the condition is not true.

Logged in or logged out

[is login]   You are logged in. [else]   You are not logged in. [/is]

Login/out

The [url] shortcode is used to display login and logout links.

Display a login link

<a href="[url login]">User login</a>

Display a logout link with redirect to home

<a href="[url logout go='home']">Logout</a>

Parameters

  • login - login link
  • logout - logout link
  • go - redirect user after login/logout; use a full URL, or the name of a post or page; go="home" will return the user to the home page.
Here is an example using both [is] and [url] to show a login/logout link based on user status. [is logout]     <a href="[url login go='home']">         Login Button     </a> [else]     <a href="[url logout go='somewhere']">         Logout Button     </a> [/is]

Redirect

The [redirect] shortcode redirects the user to another URL.

Redirect if visitor is not logged in

[is not login]     [redirect go="http://example.com/guest/"] [/is] You can also specify a relative URL by wrapping it inside. [is login]     [redirect][url site]/user-area/[/redirect] [/is] The best place to use the [redirect] shortcode is at the top of the page.
